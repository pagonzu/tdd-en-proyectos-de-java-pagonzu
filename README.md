Este repositorio contiene una implementacion practica de Test-Driven Development (TDD) para fines academicos. El objetivo principal es demostrar el flujo de trabajo Red-Green-Refactor, priorizando el diseño de pruebas antes de la escritura de codigo de produccion.

Metodologia:

Fase Roja: Creacion de pruebas unitarias fallidas basadas en requisitos.

Fase Verde: Implementacion del codigo minimo para satisfacer los tests.

Refactorizacion: Optimizacion del codigo manteniendo la integridad funcional.

Instalacion:
git clone
cd
Comando necesario para ejecutar el archivo
